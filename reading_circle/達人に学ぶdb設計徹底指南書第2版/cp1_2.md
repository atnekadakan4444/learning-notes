# 第１章

- 論理設計と物理設計はトレードオフ
- データとは「ある形式にそろえられた事実」
- 正規化は RDB でしか行われない設計手法
- データの管理構造・仕様を DB の「モデル」という
- DB のモデルによっては設計技法が異なってくるが、DBMS の違いが設計技法に影響を与える側面は薄い
- 近年は「DOA」（データ中心アプローチ）が主流
  - プログラムよりも先にデータの設計を先に始める
- 従来は「POA」（プロセス中心アプローチ）で逆が主流だった
- プログラム（処理のプロセス・ロジック）は頻繁に増減や変更があるため、プロセス中心だと DB 側も併せて増減や変更を強いられる
  - よって、DOA は「データがあまり変化しない（永続的）」ということを念頭に「プロセスの変更に柔軟に対応する」スタンス
- DOA を重視する以上、設計がアンチパターンになると致命的にシステムの品質まで影響が及ぶといえる
